Теория по ключевым моментам практической работы
1. FastAPI
Что это?
FastAPI — современный, высокопроизводительный Python-фреймворк для создания API. Он автоматически генерирует документацию (Swagger UI и ReDoc) и поддерживает асинхронность.
Зачем нужен? 
•	Для быстрого создания REST API.
•	Для автоматической валидации данных (через Pydantic).
•	Для простой генерации документации.
•	Для поддержки асинхронных операций.

2. Pydantic и модели данных
Что это?
Pydantic — библиотека для валидации данных, основанная на типизации Python. Используется в FastAPI для определения формата входящих и исходящих данных.
Зачем нужна? 
•	Для строгой типизации и проверки данных.
•	Для автоматической генерации схем в документации.
•	Для удобства при работе с JSON и другими форматами.
Пример:
from pydantic import BaseModel

class Item(BaseModel):
    name: str
    price: float
 

3. SQLModel
Что это?
SQLModel — библиотека от создателя FastAPI (Sebastián Ramírez), сочетающая возможности SQLAlchemy и Pydantic. Позволяет описывать модели данных, которые одновременно являются и ORM-моделями, и Pydantic-схемами.
Зачем нужна? 
•	Для единообразного описания моделей данных.
•	Для лучшей совместимости с FastAPI.
•	Для поддержки валидации и миграций.

4. Асинхронность (async/await)
Что это?
Асинхронность позволяет выполнять операции (например, запросы к базе данных) без блокировки выполнения других задач. Используется с ключевыми словами async def и await.
Зачем нужна? 
•	Для повышения производительности при I/O-операциях (чтение/запись в БД, сетевые запросы).
•	Для масштабируемости приложений.
Пример:
async def get_items():
    return await db.fetch_all(query)
 
5. Dependency Injection (DI)
Что это?
DI — способ передачи зависимостей (например, сессии базы данных) в функции (эндпоинты), не требуя их явного создания внутри функции.
Зачем нужно? 
•	Для упрощения тестирования.
•	Для централизованного управления ресурсами (например, сессиями БД).
•	Для чистоты кода.
Пример:
def get_session():
    with Session(engine) as session:
        yield session

@app.get("/items/")
def read_items(session: Session = Depends(get_session)):
    ...
 
6. CRUD (Create, Read, Update, Delete)
Что это?
CRUD — набор основных операций, которые позволяют управлять записями в базе данных: создание, чтение, обновление, удаление.
Зачем нужно? 
•	Для реализации стандартного функционала управления данными.
•	Для понятной архитектуры приложения.
Пример:
•	POST /items/ — Create
•	GET /items/ — Read
•	PUT /items/{id} — Update
•	DELETE /items/{id} — Delete

7. Логирование (logging)
Что это?
Механизм записи событий в приложении (например, ошибки, действия пользователя, изменения данных).
Зачем нужно? 
•	Для диагностики проблем.
•	Для отслеживания активности.
•	Для аудита и мониторинга.
Пример:
import logging
logger = logging.getLogger("uvicorn")
logger.info("Item created successfully")
 
8. Валидация данных
Что это?
Процесс проверки, соответствуют ли входящие данные заданным правилам (например, price > 0).
Зачем нужно? 
•	Для обеспечения целостности данных.
•	Для предотвращения ошибок и уязвимостей.
•	Для удобства пользователей (чёткие ошибки).
Пример:
price: float = Field(gt=0, description="Price must be positive")
